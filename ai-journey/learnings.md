# Learnings

## What worked well
- AI was fast and useful for generating a DDD folder structure and NestJS boilerplate.
- Following clean architecture principles, separation of concerns, and domain-driven design helped create a flexible structure quickly.
- Keeping test coverage high and adding pre-commit checks helped preserve context and reduced regressions while introducing new features.
- It helped draft DTO validations and Swagger annotations consistently.
- Test scaffolding with Jest and Supertest was quick to iterate on.

## What did not work
- The first iterations did not create as many test cases as requested, so additional manual test cases were necessary.
